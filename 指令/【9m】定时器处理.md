业务场景：包含多个倒计时的页面组合，比如秒杀，退款，活动开始/结束，审批时间，预计时长等，只要有多个定时器，就会产生定时器跳转不统一的问题

# 问题

- 正常的倒计时封装

```XML
<template>
    <div>{{now}}</div>
</template>
<script>
export default {
    data(){
        return {
            now:Date.now()
        }
    },
    created(){
        this.timer = setInterval(()=>{
            this.now = this.now + 1000
        },1000)
    },
    destroyed(){
        clearInterval(this.timer)
    }
}
</script> 
```

- 倒计时只有一个setTimeout或setInterval,且改属性为响应式属性，其他倒计时依赖唯一的一个倒计时属性即可

该方法用于解决倒计时多的问题以及执行顺序不统一的问题

- 使用diff代替间隔计算

因为setTimeout的精度并不确定，包括一秒一次的任务，在10s中，可能只执行了5次【上个问题】，使用更准确的diff计算，即可
